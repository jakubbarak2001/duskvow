TREE STRUCTURE — FILL IN THIS TEMPLATE EXACTLY

The tree has exactly 6 tiers. A tier is a horizontal row of nodes the player

TIER COMPOSITION (count and rarity per tier):
- Tier 1 (Foundation): 2-3 nodes, all common,    no prerequisites
- Tier 2 (Trials):     3-4 nodes, all uncommon,  each prereqs to 1-2 Tier 1 nodes

- Tier 5 (Ascension):  2-3 nodes, all legendary, each prereqs to 1-2 Tier 4 nodes
- Tier 6 (Capstone):   exactly 1 node, mythic, type = keystone,

WHY THESE RULES MATTER (do not skip any of them):
The player must always have 2+ available nodes to choose between at every
unlock point. A row with only 1 node forces a corridor and removes the entire
point of building "your own" tree. Therefore:

1. Every tier from Tier 2 through Tier 5 MUST contain at least 2 nodes.
   A Tier 2-5 row with only 1 node is a generation failure — go back and add
   a sibling.
2. No node may be the only child of its parent — every parent must have at
   least 2 children. The single exception is the Capstone keystone (Tier 6)
   which is the only one-of-its-tier node in the whole tree.
3. A node's prerequisites should usually come from the tier immediately above
   it. About one node per tier should have a SECOND prerequisite from a
   different parent in the same tier above — these are CONVERGENCE nodes
   that reward players who took both branches.
4. Use only DIRECT parents in prerequisites — never list grandparents or skip
   tiers.
5. Max 2 prerequisites per node.
